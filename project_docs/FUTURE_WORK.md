# Future Work

> 当前原型向可部署的智能座舱系统演进的可能方向。

---

## 1. 多模态疲劳感知

当前仅依赖单一 MAR 信号。人因工程中的疲劳是多维度的，需要融合：
- **EAR (Eye Aspect Ratio)**：眼睑闭合度，检测微睡眠前兆
- **PERCLOS**：单位时间内眼睛闭合百分比，工业级疲劳指标
- **头部姿态 (Head Pose)**：点头频率——疲劳的强信号
- **方向盘微操**：L2 自动驾驶中驾驶员的手部微调行为
- **生理信号**：心率变异性 (HRV)、皮肤电导 (GSR)（需额外传感器）

多模态融合后，哈欠不再独立触发。它和 EAR 下降、点头频率上升共同构成一条疲劳概率曲线。系统基于多信号置信度决定是否发起询问。

---

## 2. 个性化基线标定

当前 MAR 阈值 (0.48) 为全局固定值。不同用户面部形态差异很大：
- 嘴唇厚度影响 MAR 基线
- 自然张口习惯因人而异
- 肤色/光照影响 landmark 精度

解决方案：首次启动时 30 秒标定流程——“请保持中性表情”+“请打一个哈欠” → 计算个性化 MAR_baseline 和 MAR_threshold。

更进一步：长期标定。系统记录用户在不同时段（清晨/午后/夜间）的 MAR 分布，建立时间-唤醒度模型。

---

## 3. 自适应交互节奏

当前每次询问语相同，用户多次拒绝后仍会重复询问。改进：
- **拒绝记忆**：用户连续 N 次拒绝 Dynamic 建议 → 提升触发阈值或延长冷却
- **上下文追问**：不总是问"要不要切 Dynamic"。问法随场景变化："需要把空调调低吗？", "想听点节奏感的音乐吗？"
- **交互时机的选择**：不在驾驶员明显忙碌时发起询问（如换道、转弯、复杂路口），通过 CAN 总线获取车辆动态状态

---

## 4. LLM-based In-Car Agent

当前 NLP 是关键词匹配 + classification API。车载交互真正需要的是：
- **开放域对话理解**：驾驶员可能说任何话（"嗯...换吧", "先不用", "让我想想"）
- **上下文维护**：记住 30 秒前的交互内容
- **多轮对话**：系统可以追问、澄清、建议替代方案

技术路径：使用小参数量的本地 LLM（如 1B–3B 参数模型量化到 int4）运行在车机 SoC 上。DeepSeek API 仅作为 cloud fallback。

这不是为了好玩。LLM 在车内的真正价值是：**使系统能够处理 user agency 的表达多样性**。不是"yes/no"二选一，而是理解用户的犹豫、偏好、拒绝的轻重。

---

## 5. 非侵入式交互信道扩展

当前交互信道：TTS（输出）+ 语音（输入）。扩展方向：
- **HUD 微提示**：在挡风玻璃上投射简短建议文字，比语音更不打扰
- **触觉反馈**：方向盘振动模式变化，传递不同级别的 prompt
- **环境光微变**：在正式询问前先做微弱的氛围光试探性变化，作为 pre-cue
- **优先级分层**：低优先级建议用 HUD 图标，中优先级用触觉，高优先级用语音

原则：**信息的重要性决定信道的侵入性**。不重要的信息不应该占用高侵入性信道。

---

## 6. 长期行为学习

当前系统无记忆。每次启动都是空白。改进：
- 学习用户的模式偏好（哪些时段更倾向 Dynamic，哪些时段更倾向 Rest）
- 学习用户的拒绝模式（如果用户总是拒绝 Dynamic 建议，系统应该改变策略）
- 学习用户的哈欠-意图映射（这个用户哈欠后 80% 选择 Dynamic → 降低触发阈值；这个用户哈欠后 90% 保持 Rest → 不再询问）

数据隐私约束：行为数据不离开车辆。学习在本地完成（on-device learning）。

---

## 7. 真车部署方向

桌面原型与车规系统之间的差距：
- **硬件**：车载 SoC（如 Qualcomm Snapdragon Ride）、车规摄像头、独立声道 TTS
- **总线集成**：通过 CAN/LIN 总线读取车辆状态并控制真实空调/灯光/座椅
- **实时性**：从 Perception 到 Execution 的端到端延迟需 < 500ms
- **可靠性**：FSM 需要 watchdog + 降级策略；任一模块挂掉不影响基础驾驶功能
- **认证**：ISO 26262 功能安全、ISO 21434 网络安全

---

## 8. 研究问题

从人因工程/HCI 视角，这个方向有几个值得深入研究的问题：

1. **询问时机对用户体验的影响**：什么时间点发起询问用户接受率最高？低认知负荷时段（巡航）vs 高负荷时段（城市路口）？
2. **个性化程度与用户信任的关系**：系统越了解用户偏好，用户越信任它还是越警惕它？
3. **Shared control 中 users' mental model**：用户认为自己拥有多少控制权？他们理解系统的限制吗？
4. **询问式 vs 通知式 vs 自动执行的对比**：三种交互范式在不同场景下的用户满意度、决策质量、信任度对比实验

---

*This prototype is a starting point. The real work is in understanding how humans and intelligent vehicles can share control without either side feeling diminished.*
